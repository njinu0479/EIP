## 변수 및 배열 초기화
- 구조체 배열 a의 초기 상태:
```c
a[0] = {1, 2}
a[1] = {3, 4}
a[2] = {5, 6}
```
- ptr은 배열 a의 시작 주소(즉, &a[0])를 가리킨다.
- pptr은 포인터 변수 ptr의 주소를 가리키는 이중 포인터다.


## 핵심 포인터 연산 분석
```c
(*pptr)[1] = (*pptr)[2];
```
- *pptr은 pptr이 가리키는 곳의 값, 즉 ptr과 같다.
- 따라서 이 식은 ptr[1] = ptr[2];와 같다.
- ptr은 배열 a를 가리키고 있으므로, 결국 a[1] = a[2];라는 뜻이 된다.


## 구조체 대입의 의미
- a[1] = a[2];는 a[2]의 모든 멤버 값을 a[1]로 복사하라는 명령이다.
- 원래 a[1]은 {3, 4}였지만, 이 실행으로 인해 a[2]의 값인 {5, 6}으로 통째로 바뀐다.


## 출력 확인
```c
printf("%d 그리고 %d", a[1].x, a[1].y);
```
- 바뀌기 전의 a[1].x는 3이었지만, 현재는 5이다.
- 바뀌기 전의 a[1].y는 4였지만, 현재는 6이다.


## 최종출력결과
5 그리고 6<br/>
<br/>
⁕ 이중 포인터 해독법: (*pptr)이 나오면 그냥 원래 포인터인 ptr이라고 생각하고 한 단계를 벗겨내는 것이 요령이다.<br/>
⁕ 배열과 포인터의 호환성: ptr[i]는 *(ptr + i)와 같으며, 배열의 i번째 요소를 의미한다는 것을 잊지 말자.<br/>
* 구조체 통복사: C언어에서 구조체끼리 = 연산을 하면 내부의 모든 데이터가 한꺼번에 복사된다.
