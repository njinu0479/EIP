## 인스턴스 메서드 x(2) : 실제 객체를 따라감 (Overriding)
```java
Parent ref = new Child();
ref.x(2);
```
- 부모 타입의 리모컨(ref)으로 자식 객체(Child)를 가리키고 있다.
- 일반적인 메서드(인스턴스 메서드)는 동적 바인딩을 한다. 
- 즉, 리모컨의 종류보다 실제로 연결된 주소가 누구냐가 더 중요하다.
- 실제 객체는 Child이므로, Child에서 재정의(Overriding)한 x(int i)가 실행된다.


## 정적 메서드 id() : 참조 타입을 따라감 (Hiding)
```java
ref.id();
```
- id() 메서드는 앞부분에 static이 붙어 있다.
- 정적 메서드는 객체와 상관없이 클래스에 고정된다.
- 그래서 컴파일 시점에 객체의 타입(Parent)에 따라 어떤 메서드를 호출할지 미리 정해버린다. 이를 메서드 하이딩(Hiding)이라고 부른다.
- 리모컨(ref)의 타입이 Parent이기 때문에, 실제 객체가 Child일지라도 Parent.id()를 호출한다.


## 최종 연산 및 출력
```java
System.out.println(5 + "P");
```
5P

5P<br/>

⁕ 람다식(Lambda): (매개변수) -> { 함수몸체 } 형태를 이해해야 한다. 메서드 이름 없이 함수를 전달하는 방식이다.<br/>
⁕ 예외 처리 흐름: throw가 실행되면 즉시 해당 블록을 빠져나가 가장 가까운 catch로 점프한다는 점이 핵심이다.<br/>
⁕ 인터페이스의 기본: static interface F처럼 인터페이스 내의 추상 메서드를 람다식으로 구현하여 f라는 객체처럼 사용할 수 있다는 점을 묻고 있다.
