## 초기 상태
- 구조체 배열 a의 초기 상태:
- a -> [v: 1]
- b -> [v: 2]
- c -> [v: 3]
- arr 배열은 이 세 객체의 주소를 순서대로 담는다: [a객체, b객체, c객체]


## 배열 내 참조 교환
```c
BO t = arr[0];
arr[0] = arr[2];
arr[2] = t;
```
- 이 과정에서 a, b, c라는 원본 변수들이 가리키는 대상은 전혀 바뀌지 않았다. 
- 오직 arr 배열의 칸에 들어있는 주소들만 위치를 바꾼 것이다.
- 현재 arr 상태: [c객체, b객체, a객체]


## 값 수정
```c
arr[1].v = arr[0].v;
```
- arr[1]이 가리키는 것은 b객체이이다.
- arr[0]이 가리키는 것은 (교환된 후이므로) c객체이다.
- 즉, b객체.v = c객체.v;와 같은 뜻이 된다.
- b.v의 값이 2에서 3으로 변경된다.


## 출력 확인
```java
System.out.println(a.v + "a" + b.v + "b" + c.v);
```
- a.v: 처음 그대로 1
- c.v: 처음 그대로 3
- 따라서 문자열로 이어 붙이면 1a3b3이 된다.


## 최종출력결과
1a3b3<br/>
<br/>
⁕ arr[0] = arr[2]는 TV를 옮긴 게 아니라, 주소가 가리키는 대상만 바꾼 것이다.<br/>
⁕ 그래서 원본 변수인 a나 c가 가리키는 TV 자체에는 영향이 없다.<br/>
⁕ 스왑(Swap)이 일어난 후 arr[0]이 누구를 가리키고 있는지 차분하게 추적해야 한다.<br/>
⁕ 자바에서 숫자와 문자열을 +로 더하면 전체가 문자열로 합쳐진다.
