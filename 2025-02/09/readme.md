## run(f)의 첫 번째 호출 분석
```java
F f = (x) -> {
    if (x > 2) {
        throw new Exception();
    }
    return x * 2;
};

run(f);
```
1. run 메서드 내부에서 f.apply(3)이 실행됩니다. (인자 x는 3)
2. 람다식 내부의 조건문 if (3 > 2)가 참이 되므로 throw new Exception();이 실행된다.
3. run 메서드의 catch (Exception e) 블록이 이 예외를 낚아챈다.
4. catch 블록에서 7을 리턴한다.


## run의 두 번째 호출 분석
```java
run((int n) -> n + 9);
```
1. 이번에는 새로운 람다식 (n) -> n + 9가 run 메서드로 전달된다.
2. run 메서드 내부에서 f.apply(3)이 실행된다.
3. 계산 결과는 3 + 9 = 12가 된다.
4. 예외가 발생하지 않았으므로 try 블록에서 그대로 12를 리턴한다.


## 최종 연산 및 출력
```java
System.out.print(7 + 12);
```
7 + 12를 계산하면 19가 된다.

⁕ 람다식(Lambda): (매개변수) -> { 함수몸체 } 형태를 이해해야 한다. 메서드 이름 없이 함수를 전달하는 방식이다.<br/>
⁕ 예외 처리 흐름: throw가 실행되면 즉시 해당 블록을 빠져나가 가장 가까운 catch로 점프한다는 점이 핵심이다.<br/>
⁕ 인터페이스의 기본: static interface F처럼 인터페이스 내의 추상 메서드를 람다식으로 구현하여 f라는 객체처럼 사용할 수 있다는 점을 묻고 있다.
