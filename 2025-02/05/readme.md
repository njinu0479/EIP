## main 메서드 시작
```java
String data[] = { "A" };
```
data는 배열 객체의 주소를 가리킨다. data[0]에는 A가 들어있다.
```java
String s = "B";
```
s는 문자열 B를 가리킨다.


## change 메서드 호출
```java
change(data, s)
```
여기서 Java의 핵심 원리인 값에 의한 전달이 작동한다.<br/>
단, 참조 변수의 경우 주소값이 복사되어 전달된다.
- 배열 data: main의 data가 가리키는 똑같은 배열의 주소가 전달된다. 
- 따라서 메서드 안에서 data[0]을 바꾸면 진짜 원본 배열이 바뀐다.
- 문자열 s: main의 s가 가진 주소값(B의 위치)이 전달된다.


## change 메서드 내부 실행
```java
data[0] = s;
s = "Z";
```
- data[0] = s;: data가 가리키는 실제 배열의 0번 칸에 현재 전달받은 s의 값인 B를 넣는다.
- s = "Z";: 매개변수로 넘어온 지역 변수 s에게 "Z"라는 새로운 주소를 할당한다.
- 이 s는 main에 있는 s와는 이름만 같을 뿐, 복사된 별개의 변수이다. 
- 따라서 이 변수가 다른 곳을 가리키게 한다고 해서 main의 변수(s)가 가리키는 방향이 바뀌지는 않는다.


## 최종출력결과
BB<br/>
<br/>
⁕ 배열(Object): 메서드에 인자로 넘겨서 내부의 값을 바꾸면 원본이 바뀐다.<br/>
⁕ String/기본형: 메서드 내부에서 값을 재할당(s = "Z")해도 원본에는 아무런 영향이 없다.
