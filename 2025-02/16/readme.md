## 초기 상태
- a: {p: 1, n: NULL}
- b: {p: 2, n: NULL}
- c: {p: 3, n: NULL}


## 배열 내 참조 교환
```c
BO t = arr[0];
arr[0] = arr[2];
arr[2] = t;
```
- 이 과정에서 a, b, c라는 원본 변수들이 가리키는 대상은 전혀 바뀌지 않았다. 
- 오직 arr 배열의 칸에 들어있는 주소들만 위치를 바꾼 것이다.
- 현재 arr 상태: [c객체, b객체, a객체]


## 첫 번째 연결
```c
a.n = &b; b.n = &c; c.n = NULL;
```
- 연결 순서: a(1) → b(2) → c(3) → NULL


## 두 번째 연결
```c
c.n = &a; a.n = &b; b.n = NULL;
```
- c.n = &a; : c가 a를 가리킨다. (c → a)
- a.n = &b; : a가 b를 가리킨다. (a → b)
- b.n = NULL; : b는 아무것도 가리키지 않는다. (b → NULL)
- 최종 연결 고리: c(3) → a(1) → b(2) → NULL


## 출력 확인
- struct node* head = &c; 이므로 head는 c를 가리키고 있다.
1. head->p: c의 p 값 -> 3
2. head->n->p: c가 가리키는 노드(a)의 p 값 -> 1
3. head->n->n->p: c가 가리키는 노드(a)가 가리키는 노드(b)의 p 값 -> 2
따라서 출력은 3 1 2가 된다.

## 최종출력결과
3 1 2<br/>
<br/>
⁕ C언어는 위에서 아래로 실행되기 때문에 같은 변수에 값을 두 번 대입하면 나중에 대입한 값이 살아남는다.<br/>
⁕ 포인터 문제는 눈으로만 풀면 헷갈리기 쉽다. 시험지 여백에 c -> a -> b 처럼 화살표를 직접 그리면서 푸는 것이 가장 정확하다.
