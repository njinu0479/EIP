## 데이터의 배치 (set 함수)
- 이 코드에서 가장 어려운 부분은 data 배열의 값들이 2차원 배열 arr의 어디로 들어가는지 찾는 것이다.
- 인덱스 계산식에 +1이 붙어 있어서 데이터가 한 칸씩 밀려 들어가는 것이 핵심이다.
- 행(Row): ((i + 1) / 3) % 3
- 열(Col): (i + 1) % 3

| i | data[i] | i + 1 | 행(Row) 계산 | 열(Col) 계산 | 저장 위치 |
| :---: | :---: | :---: | :---: | :---: | :---: |
| 0 | 5 | 1 | (1 / 3) % 3 = 0 | 1 % 3 = 1 | arr[0][1] |
| 1 | 2 | 2 | (2 / 3) % 3 = 0 | 2 % 3 = 2 | arr[0][2] |
| 2 | 7 | 3 | (3 / 3) % 3 = 1 | 3 % 3 = 0 | arr[1][0] |
| 3 | 4 | 4 | (4 / 3) % 3 = 1 | 4 % 3 = 1 | arr[1][1] |
| 4 | 1 | 5 | (5 / 3) % 3 = 1 | 5 % 3 = 2 | arr[1][2] |
| 5 | 8 | 6 | (6 / 3) % 3 = 2 | 6 % 3 = 0 | arr[2][0] |
| 6 | 3 | 7 | (7 / 3) % 3 = 2 | 7 % 3 = 1 | arr[2][1] |
| 7 | 6 | 8 | (8 / 3) % 3 = 2 | 8 % 3 = 2 | arr[2][2] |
| 8 | 9 | 9 | (9 / 3) % 3 = 0 | 9 % 3 = 0 | arr[0][0] |

## 완성된 2차원 배열의 모습
- 위의 계산 결과에 따라 3x3 배열은 다음과 같은 상태가 된다. 
- 1차원 데이터가 0,0부터 들어가는 게 아니라 0,1부터 시작해서 한 바퀴 돌아 0,0에서 끝나는 형태이다.

| | 열(0) | 열(1) | 열(2) |
| :---: | :---: | :---: | :---: |
| 행(0) | 9 | 5 | 2 |
| 행(1) | 7 | 4 | 1 |
| 행(2) | 8 | 3 | 6 |

## 최종 합계(sum) 계산
- 마지막으로 arr의 값을 0,0부터 2,2까지 순서대로 꺼내며 더하거나 뺀다.
- 규칙: 인덱스(i)가 0, 2, 4, 6, 8(짝수)이면 더하고(+1), 1, 3, 5, 7(홀수)이면 뺀다(-1).
- 계산 과정
1. i=0: arr[0][0]인 9 (짝수 -> +9)
2. i=1: arr[0][1]인 5 (홀수 -> -5)
3. i=2: arr[0][2]인 2 (짝수 -> +2)
4. i=3: arr[1][0]인 7 (홀수 -> -7)
5. i=4: arr[1][1]인 4 (짝수 -> +4)
6. i=5: arr[1][2]인 1 (홀수 -> -1)
7. i=6: arr[2][0]인 8 (짝수 -> +8)
8. i=7: arr[2][1]인 3 (홀수 -> -3)
9. i=8: arr[2][2]인 6 (짝수 -> +6)

- 최종 연산: 9 - 5 + 2 - 7 + 4 - 1 + 8 - 3 + 6 = 13

⁕ arr[i / rows][i % cols]는 2차원 배열을 1차원처럼 순서대로 훑을 때 쓰는 전형적인 공식이다.<br/>
⁕ (i % 2 == 0 ? 1 : -1)는 부호를 교대로 바꾸는 기술이다.<br/>
⁕ malloc으로 할당한 배열은 free를 통해 역순으로 해제해주는 것이 정석이다.
