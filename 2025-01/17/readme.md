## 트리 구조 파악하기 (tree 함수)
- 리스트 li = [3, 5, 8, 12, 15, 18, 21]를 바탕으로 부모-자식 관계를 맺어준다.
- 공식 (i - 1) // 2는 배열로 구현된 이진 트리에서 부모의 인덱스를 찾는 공식이다.
- i=1, 2: 부모는 (0)번 노드인 3 -> 자식은 5, 8
- i=3, 4: 부모는 (1)번 노드인 5 -> 자식은 12, 15
- i=5, 6: 부모는 (2)번 노드인 8 -> 자식은 18, 21

```
      3 (Level 0)
     / \
    5   8 (Level 1)
   / \ / \
  12 15 18 21 (Level 2)
```


## 레벨별 합계 계산 (calc 함수)
- calc 함수는 재귀적으로 모든 노드를 방문하면서 값을 더하는데, 조건이 붙어 있다.
- node.value if level % 2 == 1 else 0 -> 즉, 레벨이 홀수(1, 3, 5...)인 노드의 값만 더하라는 뜻이다.
- Level 0 (짝수): 노드 3 -> 더하지 않음 (0)
- Level 1 (홀수): 노드 5, 8 -> 5 + 8 = 13
- Level 2 (짝수): 노드 12, 15, 18, 21 -> 더하지 않음 (0)
- 최종 결과는 0 + 13 + 0 = 13이 된다.


---
⁕ i번째 노드의 왼쪽 자식은 2i + 1, 오른쪽 자식은 2i + 2, 부모는 (i-1)//2라는 인덱스 규칙은 다른 언어도 공통으로 쓰이는 공식.
⁕ 보통 루트 노드를 Level 0으로 잡는다. 문제에서 level=0으로 초기값을 줬으므로 이를 기준으로 짝/홀을 판단해야 한다.
⁕ sum(calc(...) for n in node.children)은 자식 노드들이 반환한 모든 값을 합치겠다는 의미이다.
