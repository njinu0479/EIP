## 예외 발생
- int a=5, b=0; 상태에서 a / b 즉, 5/0 연산을 시도한다.
- 정수를 0으로 나누면 자바에서는 ArithmeticException (산술 연산 예외)이 발생한다.


## catch 블록 탐색
- 첫 번째 catch (ArithmeticException): 현재 발생한 예외와 정확히 일치한다.
- 나머지 catch들: 자바의 예외 처리는 위에서부터 내려오며 가장 먼저 일치하는 하나만 실행하고 나머지는 건너뛴다.


## finally 블록 실행
- 예외가 발생했든 안 했든, 잡혔든 안 잡혔든 무조건 실행되는 블록이다.


## 최종출력결과
출력1출력5<br/>
<br/>
⁕ 상세한 예외(ArithmeticException)를 위에 쓰고, 포괄적인 예외(Exception)를 아래에 써야한다.<br/>
⁕ 만약 Exception이 맨 위에 있다면 모든 예외를 얘가 다 가로채기 때문에 아래쪽 코드들은 실행될 기회가 없어 컴파일 에러가 날 수 있다.<br/>
⁕ try 블록이나 catch 블록 안에 return이 있어도 finally는 기어코 자기 할 일을 하고 나서야 메서드를 종료시킨다.<br/>
⁕ print 함수를 쓰고 있으므로 줄바꿈 없이 출력1출력5라고 붙여서 적어야 정확한 정답이다.
